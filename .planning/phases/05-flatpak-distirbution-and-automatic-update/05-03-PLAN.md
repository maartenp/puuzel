**1. .github/workflows/release.yml** (per D-12):

```yaml
name: Release

on:
  push:
    tags: ['v*']

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    container:
      image: ghcr.io/andyholmes/flatter/freedesktop:24.08
      options: --privileged
    steps:
      - uses: actions/checkout@v4

      - name: Install flatpak-cargo-generator dependencies
        run: pip install aiohttp toml

      - name: Clone flatpak-builder-tools
        run: git clone --depth 1 https://github.com/flatpak/flatpak-builder-tools.git

      - name: Generate cargo-sources.json from Cargo.lock
        run: |
          python3 flatpak-builder-tools/cargo/flatpak-cargo-generator.py \
            Cargo.lock -o flatpak/cargo-sources.json

      - name: Write version.txt for in-app update check
        run: echo "${{ github.ref_name }}" | sed 's/^v//' > version.txt

      - name: Build Flatpak
        id: flatter
        uses: andyholmes/flatter@main
        with:
          files: flatpak/io.github.maartenp.puuzel.yml
          upload-pages-artifact: true

  deploy:
    needs: build
    runs-on: ubuntu-latest
    permissions:
      pages: write
      id-token: write
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

Key decisions:
- Uses `ghcr.io/andyholmes/flatter/freedesktop:24.08` image (matches SDK 24.08, not gnome:master which may have different SDK version)
- `cargo-sources.json` generated in CI per D-13 (not committed)
- `version.txt` written before the flatter step — flatter's `upload-pages-artifact` should include it since it's in the workspace. If not, a workaround step may be needed (see open question in research). The executor should verify this works on first release and adjust if version.txt is missing from gh-pages.
- Permissions set at workflow level for pages deployment
- `pip install aiohttp toml` — the generator script requires these Python packages (note: some versions use `toml` instead of `tomlkit`)
