  <action>
Create `release.sh` at repo root (per D-11). Must be executable (`chmod +x`).

Script behavior:
1. `set -euo pipefail`
2. Read current version: `grep '^version' Cargo.toml | head -1 | sed 's/.*"\(.*\)".*/\1/'`
3. Print: `echo "Current version: $current"`
4. Prompt: `read -rp "Increment [M]ajor / [m]inor / [p]atch? " choice`
5. Parse `$current` into MAJOR.MINOR.PATCH using IFS='.'
6. Increment based on choice:
   - `M` or `major`: MAJOR+1, MINOR=0, PATCH=0
   - `m` or `minor`: MINOR+1, PATCH=0
   - `p` or `patch`: PATCH+1
   - else: print "Invalid choice" and exit 1
7. Construct `new_version="$MAJOR.$MINOR.$PATCH"`
8. Print: `echo "New version: $new_version"`
9. Confirm: `read -rp "Confirm? [y/N] " confirm` — exit 0 if not 'y'
10. `sed -i '' "s/^version = \"$current\"/version = \"$new_version\"/" Cargo.toml` (macOS sed uses `-i ''`, add Linux fallback with `sed -i`)
11. Use portable sed: detect OS and use appropriate -i flag (`sed -i ''` on macOS, `sed -i` on Linux)
12. `git add Cargo.toml Cargo.lock`
13. `cargo check` to update Cargo.lock with new version
14. `git add Cargo.toml Cargo.lock`
15. `git commit -m "chore: bump version to $new_version"`
16. `git tag "v$new_version"`
17. `git push && git push --tags`

Also verify DIST-04: run `grep 'source = "git+' Cargo.lock` and confirm zero matches. If any match, log a warning.
  </action>
