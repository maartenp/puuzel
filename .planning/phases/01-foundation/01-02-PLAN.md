    `pub fn is_connected(grid: &Grid) -> bool`:
    - Find the first white cell in the grid (scan row by row)
    - If no white cell exists, return true (degenerate case)
    - BFS/flood-fill from that cell, marking visited white cells
    - After BFS completes, check that all white cells in the grid are visited
    - Return true if all white cells reached, false otherwise
    - Use a `Vec<Vec<bool>>` for the visited matrix (20x20 = 400 bools, no allocation concern)
    - Directions: up, down, left, right (4-connected)

    Create src/grid/difficulty.rs:

    `pub fn seed_black_squares(grid: &mut Grid, config: &DifficultyConfig, rng: &mut impl rand::Rng)`:
    - Target black square count = random float in [config.black_square_ratio_min, config.black_square_ratio_max] * (width * height), rounded to nearest int
    - Phase 1 — Random placement: place black squares one at a time at random white positions
    - After each placement, check:
      1. No isolated white cell created (check 4 neighbors of all adjacent white cells — each must have at least 1 white orthogonal neighbor)
      2. No 2x2 all-white block is required to be absent — actually check that we haven't created a configuration where the only way to break a 2x2 white block requires more black squares than our budget allows. Simpler: just disallow 2x2 all-white blocks in final validation.
    - If a placement violates isolation, undo it and try another position
    - Phase 2 — Connectivity validation: after reaching target count, run is_connected(). If disconnected, remove the last-placed black square and try a different position.
    - Phase 3 — Slot validation: scan for word slots (consecutive white cells in a row or column). If any slot has length < 2, convert that single white cell to black (it would be an isolated letter).
    - Phase 4 — 2x2 white block check: scan all 2x2 subgrids. If any has all 4 cells white, place a black square in one of the 4 positions (choose the one that causes least disruption — prefer the one with most black neighbors).
    - Retry from scratch (up to 10 attempts) if constraints cannot be satisfied.

    `pub fn extract_slots(grid: &Grid) -> Vec<Slot>`:
    - Scan each row left-to-right: sequences of 2+ consecutive white cells form Across slots
    - Scan each column top-to-bottom: sequences of 2+ consecutive white cells form Down slots
    - Return all slots as Vec<Slot>
    - Each slot records its starting (row, col), direction, and length

    Create src/grid/generator.rs with the slot-based CSP approach:

    **Data structures:**

    `WordIndex` — pre-built lookup for fast candidate filtering:
    - `by_length: HashMap<usize, Vec<(i64, Vec<LetterToken>)>>` — words grouped by grid_length, each stored as (word_id, tokens)
    - `by_length_and_pos: HashMap<(usize, usize, LetterToken), Vec<usize>>` — maps (word_length, position_index, required_token) to indices into by_length[word_length]. This is the position-letter index from the research.
    - `fn build(conn: &Connection, config: &DifficultyConfig) -> Result<Self>`: Load words from DB using words_for_length for lengths 2 through 15 with config.min_commonness. Tokenize each word. Build both indexes.

    `SlotState` — tracking per-slot during generation:
    - `slot: Slot`
    - `assigned_word: Option<(i64, Vec<LetterToken>)>` — word_id and tokens
    - `constraints: Vec<(usize, LetterToken)>` — (position_in_slot, required_token) from crossing words already placed

    `FilledGrid` struct:
    - `grid: Grid` — the grid with all letters filled in
    - `slot_words: Vec<(Slot, i64)>` — which word_id was placed in each slot
    - `difficulty: Difficulty`

    **Core algorithm — `pub fn generate_grid(conn: &Connection, config: &DifficultyConfig) -> Result<FilledGrid, GeneratorError>`:**

    1. Create a 20x20 Grid, call `seed_black_squares(&mut grid, config, &mut rng)` (per D-01, all difficulties use 20x20)
    2. Validate connectivity with `is_connected(&grid)` — if false, retry seeding (up to 10 times)
    3. Call `extract_slots(&grid)` to get all word positions
    4. Build `WordIndex` from the database
    5. Initialize `Vec<SlotState>` from slots, no constraints yet
    6. Build a crossing map: for each pair of slots that share a cell, record which position in slot A corresponds to which position in slot B
    7. **CSP loop with MRV (Most Restricted Variable) heuristic:**
       - Pick the unassigned slot with the fewest valid candidates (given current constraints)
       - If no candidates exist for any unassigned slot, backtrack
       - Shuffle candidates randomly (for variety between generations)
       - Try each candidate word:
         a. Assign word to slot (write LetterTokens into SlotState)
         b. Propagate constraints to all crossing slots (add (position, token) constraints)
         c. Check that all crossing slots still have at least 1 valid candidate (forward checking)
         d. If forward checking passes, recurse to next slot
         e. If forward checking fails or recursion fails, undo assignment and constraints, try next candidate
       - If all candidates exhausted, return failure (backtrack to caller)
    8. Set a timeout of 8 seconds using `std::time::Instant`. If exceeded, return `GeneratorError::Timeout`.
    9. On success, write all assigned LetterTokens into Grid cells, validate is_connected one final time, return FilledGrid.
    10. Track used word_ids to avoid duplicates within a single grid.

    `pub enum GeneratorError`:
    - `Timeout` — generation exceeded 8 seconds
    - `NoSolution` — exhausted search space
    - `DatabaseError(rusqlite::Error)` — word loading failed

    Implement `std::fmt::Display` for GeneratorError.

    **Performance notes:**
    - The MRV heuristic is critical: always fill the most constrained slot first to prune early
    - Forward checking (step 7c) prevents exploring dead-end branches
    - Shuffling candidates ensures different puzzles each time
    - 8-second timeout prevents infinite loops on tight grids (Pitfall 1)
