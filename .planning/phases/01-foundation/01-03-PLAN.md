    Create tools/filter_wordlist.py:

    The script does the following when run as `python tools/filter_wordlist.py`:

    1. **Download OpenTaal**: Download `basiswoorden-gekeurd.txt` from the OpenTaal GitHub repo raw URL: `https://raw.githubusercontent.com/OpenTaal/opentaal-wordlist/master/elements/basiswoorden-gekeurd.txt`. Save to `tools/cache/basiswoorden-gekeurd.txt`. Skip download if file already exists.

    2. **Load blocklist**: Read `tools/dutch_blocklist.txt` into a set (lowercase).

    3. **Filter each word**:
       - Strip whitespace
       - Skip empty lines
       - Compute grid_length using IJ normalization:
         - Replace U+0132 with "IJ", U+0133 with "ij"
         - Uppercase
         - Count tokens: "IJ" = 1 token, all other chars = 1 token each
       - Exclude if grid_length < 2 or grid_length > 15 (D-09, D-10)
       - Exclude if word contains '.' (abbreviations, D-07)
       - Exclude if word.lower() in blocklist (D-11)
       - Exclude if word contains digits
       - Exclude if word contains spaces or hyphens (compound separators — compounds without separators are fine per D-13)

    4. **Detect proper nouns**: If the original word starts with an uppercase letter and is not all-uppercase, mark `is_proper_noun: true` (per D-08, proper nouns are allowed).

    5. **Output**: Write to `tools/output/filtered_words.json` as a JSON array:
       ```json
       [{"word": "IJSBEER", "grid_length": 6, "is_proper_noun": false}, ...]
       ```
       All words stored in uppercase.

    6. **Print summary**: Total words read, words after filter, proper noun count.

    Make the filtering logic available as importable functions (not just in `if __name__ == "__main__"` block):
    - `compute_grid_length(word: str) -> int`
    - `is_abbreviation(word: str) -> bool`
    - `filter_word(word: str, blocklist: set) -> bool` — returns True if word should be KEPT
    - `normalize_word(word: str) -> str` — uppercase + Unicode IJ normalization

    Create tools/generate_clues.py:

    The script runs as `python tools/generate_clues.py [--test-batch N]`:

    **Step 1: Load filtered words**
    - Read `tools/output/filtered_words.json`
    - If `--test-batch N` flag, take only first N words (for testing with small batch)

    **Step 2: Create generation batch**
    - For each word, create a batch request using the Anthropic Python SDK:
      ```python
      {
          "custom_id": f"gen-{word_upper}",
          "params": {
              "model": "claude-haiku-4-5-20250315",
              "max_tokens": 300,
              "messages": [{
                  "role": "user",
                  "content": f"""Genereer drie aanwijzingen in het Nederlands voor het kruiswoordraadselwoord "{word}".

    Regels:
    - Geen woordspeling, cryptische of dubbelzinnige aanwijzingen
    - Maximaal 6 woorden per aanwijzing
    - Makkelijk: directe omschrijving of synoniem
    - Middel: feitelijk maar meer nadenken vereist
    - Moeilijk: indirect, vraagt specifieke kennis of inferentie
    - Als het woord ouderwets of literair is, begin de moeilijke aanwijzing met "Ouderwets woord voor" (per D-12)
    - Geef een gewoonheidsscore: 1=zeldzaam/vakjargon, 2=ongewoon, 3=bekend, 4=veelgebruikt, 5=alledaags
    - Geef is_archaic: true als het woord ouderwets of louter literair is, anders false

    Antwoord ALLEEN in dit JSON-formaat, geen andere tekst:
    {{"easy": "...", "medium": "...", "hard": "...", "commonness": 3, "is_archaic": false}}"""
              }]
          }
      }
      ```
    - Submit batch via `client.messages.batches.create(requests=batch_requests)`
    - Poll for completion: `client.messages.batches.retrieve(batch_id)` until status is "ended"
    - Parse results, extract JSON from each response. Extract `is_archaic` boolean (default false if missing).
    - Save generation results to `tools/output/generated_clues.json`

    **Step 3: Create verification batch**
    - For each generated clue (3 per word = 3 verification requests per word):
      ```python
      {
          "custom_id": f"verify-{word_upper}-{difficulty}",
          "params": {
              "model": "claude-haiku-4-5-20250315",
              "max_tokens": 50,
              "messages": [{
                  "role": "user",
                  "content": f"Los dit kruiswoordraadsel op. Geef ALLEEN het antwoord als een enkel Nederlands woord, niets anders.\n\nAanwijzing: {clue_text}"
              }]
          }
      }
      ```
    - Submit verification batch, poll for completion
    - For each verification response:
      - Normalize both the target word and the answer (uppercase, strip whitespace)
      - Mark clue as `verified: true` if answer matches target word exactly
      - Mark clue as `verified: false` if answer does not match
    - Save verification results to `tools/output/verified_clues.json` as:
      ```json
      [{"word": "IJSBEER", "grid_length": 6, "commonness": 4, "is_proper_noun": false, "is_archaic": false, "clues": [
        {"difficulty": "easy", "text": "Wit poolroofdier", "verified": true},
        {"difficulty": "medium", "text": "Arctisch zoogdier", "verified": true},
        {"difficulty": "hard", "text": "Ursus maritimus", "verified": false}
      ]}]
      ```
      Note: `is_archaic` is a top-level word-level field (not per-clue). It is the `is_archaic` value returned by the LLM in the generation response.

    **Step 4: Print summary**
    - Total words processed, clues generated, verification pass rate per difficulty level
    - Warn if any difficulty has < 70% pass rate
    - Print count of words flagged as archaic

    **Error handling:**
    - If a batch request fails, log the error and skip that word
    - If JSON parsing fails for a response, log and skip
    - Save progress incrementally (write partial results after each batch completes)

    Create tools/write_database.py:

    The script runs as `python tools/write_database.py`:

    1. Read `tools/output/verified_clues.json`
    2. Create SQLite database at `data/puuzel.db`
    3. Create schema (same as Rust db/schema.rs):
       ```sql
       CREATE TABLE IF NOT EXISTS words (
           id INTEGER PRIMARY KEY,
           word TEXT NOT NULL UNIQUE,
           grid_length INTEGER NOT NULL,
           commonness_score INTEGER NOT NULL,
           is_proper_noun INTEGER NOT NULL DEFAULT 0,
           is_archaic INTEGER NOT NULL DEFAULT 0
       );
       CREATE TABLE IF NOT EXISTS clues (
           id INTEGER PRIMARY KEY,
           word_id INTEGER NOT NULL REFERENCES words(id),
           difficulty TEXT NOT NULL CHECK(difficulty IN ('easy', 'medium', 'hard')),
           clue_text TEXT NOT NULL,
           verified INTEGER NOT NULL DEFAULT 0,
           thumbs_down INTEGER NOT NULL DEFAULT 0
       );
       CREATE INDEX IF NOT EXISTS idx_words_grid_length ON words(grid_length);
       CREATE INDEX IF NOT EXISTS idx_words_commonness ON words(commonness_score);
       CREATE INDEX IF NOT EXISTS idx_clues_word_difficulty ON clues(word_id, difficulty);
       CREATE INDEX IF NOT EXISTS idx_clues_verified ON clues(verified);
       ```
    4. For each word in verified_clues.json:
       - INSERT word into words table: commonness from LLM rating, is_proper_noun from filter output, is_archaic from LLM `is_archaic` field (1 if true, 0 if false or absent)
       - INSERT each clue into clues table with verified flag
       - Only insert clues where verified=true
    5. Print summary: words inserted, clues inserted per difficulty, archaic word count, total DB size
    6. Run validation queries:
       - `SELECT COUNT(*) FROM words` — print count
       - `SELECT difficulty, COUNT(*) FROM clues WHERE verified=1 GROUP BY difficulty` — print per-difficulty counts
       - `SELECT grid_length, COUNT(*) FROM words GROUP BY grid_length ORDER BY grid_length` — print length distribution
       - `SELECT COUNT(*) FROM words WHERE is_archaic=1` — print archaic word count
